# 项目架构

## 整体架构

本项目采用 Monorepo 的方式组织代码，使用 Vue 3 + TypeScript + Vite 作为前端技术栈，基于 Ant Design Vue 组件库进行二次封装，提供一套完整的企业级中后台前端解决方案。

### 项目结构

```
vue-vben-admin/
├── apps/                       # 应用目录
│   ├── web-antd/               # 基于 Ant Design Vue 的前端应用
│   ├── backend-mock/           # 模拟后端服务
│   └── docs/                   # 项目文档
│
├── packages/                   # 共享包目录
│   ├── @core/                  # 核心工具和组件
│   │   ├── icons/              # 图标库
│   │   ├── shared/             # 共享工具函数
│   │   ├── typings/            # TypeScript 类型定义
│   │   └── ui-kit/             # UI 基础组件
│   │
│   ├── @vben/                  # 业务组件和工具
│       ├── common-ui/          # 通用 UI 组件
│       ├── stores/             # 状态管理
│       └── types/              # 业务类型定义
│
└── README.md                   # 项目说明
```

## 前端架构

### 技术栈

- **框架**：Vue 3 + TypeScript
- **构建工具**：Vite
- **UI组件**：Ant Design Vue
- **状态管理**：Pinia
- **路由**：Vue Router
- **HTTP客户端**：Axios
- **表格组件**：VXE-Table
- **表单组件**：FormKit (自定义适配)
- **工具库**：VueUse

### 核心设计理念

1. **组件化开发**：将UI和功能拆分为独立、可复用的组件
2. **配置式开发**：通过配置生成表单、表格等，减少重复代码
3. **约定大于配置**：遵循一致的开发规范，提高开发效率
4. **类型安全**：全面使用TypeScript，提供类型检查和智能提示
5. **插件化设计**：核心功能以插件形式提供，按需加载

## 数据流

### 前端数据流

1. **用户操作** → **事件处理** → **API调用** → **状态更新** → **视图更新**

2. **表单数据流**：

   - 通过Schema配置生成表单
   - 用户输入触发表单值更新
   - 表单提交时收集数据并调用API
   - 根据API返回结果更新UI状态

3. **表格数据流**：
   - 页面加载或筛选条件变更触发数据加载
   - 通过ProxyConfig定义数据获取逻辑
   - API返回数据后更新表格视图
   - 分页、排序等操作触发新的数据加载

### 状态管理

使用Pinia进行状态管理，按照功能模块划分Store：

1. **用户状态**：管理用户信息、权限等
2. **应用状态**：管理应用全局设置、主题等
3. **菜单状态**：管理菜单配置和权限
4. **标签页状态**：管理多标签页功能

## 权限控制

### 权限模型

采用基于角色(RBAC)的权限控制模型：

1. **菜单权限**：控制菜单的显示和访问
2. **按钮权限**：控制页面中按钮等操作的显示和使用
3. **数据权限**：控制用户可访问的数据范围

### 权限实现

1. **前端路由守卫**：验证用户是否有权限访问页面
2. **动态菜单**：根据用户权限动态生成菜单
3. **权限指令**：使用自定义指令控制按钮等元素的显示
4. **接口权限**：API调用时携带权限信息，后端验证

## 国际化

使用 `i18n` 实现国际化，支持中英文切换：

1. **文本国际化**：页面文本、提示信息等使用 `$t` 函数包装
2. **组件国际化**：表单标签、表格列标题等配置中使用 `$t` 函数
3. **日期时间国际化**：根据语言环境格式化日期时间

## 主题定制

支持主题定制，可以调整：

1. **主题色**：主要颜色、强调色等
2. **暗色/亮色模式**：支持自动切换或手动设置
3. **布局选项**：侧边栏位置、宽度、菜单展开方式等

## 最佳实践

1. **模块化设计**：功能模块独立，降低耦合
2. **类型安全**：充分利用TypeScript类型系统
3. **组件封装**：将通用功能封装为组件，提高复用性
4. **配置式开发**：使用配置生成UI，减少重复代码
5. **标准化错误处理**：统一的错误捕获和展示机制
6. **性能优化**：路由懒加载、组件异步加载、虚拟滚动等
