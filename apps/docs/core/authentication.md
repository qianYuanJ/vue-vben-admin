# 认证与授权模块

## 模块概述

认证与授权模块是系统的核心安全组件，负责用户登录认证、权限验证和安全退出等功能。该模块确保只有授权用户才能访问系统资源，并根据用户角色提供相应的访问权限。

## 页面结构

- **登录页面** - `apps/web-antd/src/views/_core/authentication/login/index.vue`

  - 提供用户名密码登录表单
  - 支持验证码登录选项
  - 包含记住密码功能
  - 提供忘记密码入口

- **验证码登录** - `apps/web-antd/src/views/_core/authentication/code-login/index.vue`

  - 基于手机号和验证码的登录方式
  - 包含验证码发送和倒计时功能

- **退出登录处理** - 通过路由守卫和状态管理实现

## 数据交互

### API接口

- 用户名密码登录：`POST /api/auth/login`
- 验证码登录：`POST /api/auth/login/mobile`
- 获取验证码：`GET /api/auth/code`
- 刷新令牌：`POST /api/auth/refresh-token`
- 登出：`POST /api/auth/logout`
- 获取用户权限信息：`GET /api/auth/access-codes`

### 数据流程

1. **登录流程**：

   - 用户输入凭证（用户名/密码或手机号/验证码）
   - 前端进行表单验证
   - 调用登录API，获取访问令牌和刷新令牌
   - 存储令牌到状态管理和本地存储
   - 获取用户信息和权限信息
   - 重定向到系统首页或指定页面

2. **权限验证流程**：

   - 路由跳转前，检查用户是否已登录
   - 验证用户是否有权限访问目标路由
   - 根据验证结果允许访问或重定向到登录页

3. **退出流程**：
   - 调用登出API通知后端
   - 清除前端存储的令牌和用户信息
   - 重定向到登录页面

## 关键实现

### 令牌管理

系统采用JWT（JSON Web Token）方案进行身份验证：

1. **访问令牌（Access Token）**：短期有效，用于API请求认证
2. **刷新令牌（Refresh Token）**：长期有效，用于获取新的访问令牌
3. **令牌刷新机制**：访问令牌过期时，自动使用刷新令牌获取新的访问令牌

### 请求拦截器

使用Axios拦截器实现：

1. 为每个请求自动添加认证头（Bearer Token）
2. 捕获401错误，尝试刷新令牌并重试请求
3. 处理刷新令牌失败的情况，引导用户重新登录

### 路由守卫

通过Vue Router的全局前置守卫实现：

1. 检查路由的访问权限要求
2. 验证用户是否已登录
3. 检查用户权限是否满足路由要求
4. 根据验证结果决定允许访问或重定向

## 安全考虑

1. **密码安全**：

   - 前端传输前进行加密处理
   - 使用HTTPS确保传输安全
   - 密码规则验证，确保强度

2. **令牌安全**：

   - 短生命周期的访问令牌
   - 令牌存储在内存状态中，减少XSS风险
   - 支持令牌吊销机制

3. **防护措施**：
   - 登录尝试限制，防止暴力破解
   - CSRF防护
   - XSS防护

## 注意事项

1. 登录状态保持应注意安全与便利的平衡
2. 前端权限控制只能防止普通用户的误操作，核心安全控制应在后端实现
3. 敏感操作应要求用户重新验证身份
4. 多终端登录策略应与业务需求保持一致，可配置为互斥登录或多设备同时在线
