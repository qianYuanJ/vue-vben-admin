# 项目文档中心

本目录用于存放项目各模块的详细说明文档，帮助开发人员了解项目架构和各模块的功能实现。

## 目录结构

- `/system` - 系统管理模块相关文档
  - `department.md` - 部门管理模块说明
  - `employee.md` - 员工管理模块说明
  - `role.md` - 角色权限管理模块说明
  - `operation-log.md` - 操作日志模块说明
  - `app-version.md` - APP版本管理模块说明
- `/dashboard` - 仪表盘模块相关文档
  - `overview.md` - 仪表盘模块概述
- `/core` - 核心功能模块相关文档
  - `authentication.md` - 认证与授权模块说明
  - `fallback.md` - 错误处理与回退页面模块说明
  - `about.md` - 关于页面模块说明
- `/demos` - 演示模块相关文档
  - `antd.md` - Ant Design Vue 组件演示模块说明
- `/common` - 公共组件和功能说明文档
  - `components.md` - 公共组件说明
  - `architecture.md` - 项目架构说明

## 文档编写规范

每个模块文档应包含以下内容：

1. 模块概述：简要说明模块的功能和用途
2. 页面结构：描述模块包含的页面和各页面的功能
3. 数据交互：说明模块与后端的数据交互方式和API接口
4. 关键实现：记录模块的核心实现逻辑和特殊处理
5. 注意事项：使用过程中需要注意的问题和限制

## 使用方法

开发新模块时，请参考已有模块的实现方式和文档规范，并及时更新对应的文档说明。文档应与代码同步更新，确保文档的准确性和时效性。

## 文档维护

1. 当添加新功能或修改现有功能时，应同步更新相关文档
2. 文档中的示例代码应确保可以正常运行
3. 保持文档格式统一，遵循Markdown语法规范
4. 重要的更新应在文档修改历史中记录
